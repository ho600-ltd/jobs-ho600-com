一般企業常見的生日禮金、三節禮金、員工旅遊、尾牙抽獎都不會有，主要原因是敝公司不是「一般企業」，而次要原因如下：

  • 生日禮金： 創辦人自己都不過生日了，還發什麼生日禮金，不如就把這幾千元禮金攤進薪資福利就好
  • 三節禮金： 敞公司平等對待不同種族、階級、語言、思想、宗教、黨派、籍貫、出生地、性別、性傾向、年齡、婚姻、容貌、五官、身心障礙或以往工會會員身分者，這三節太偏閩客中華文化了，發三節禮金，那對原住民員工而言，要改發什麼節禮金? 不如就把這幾千元禮金攤進薪資福利就好
  • 員工旅遊： 敝公司聘請員工，主要目的是賺錢，不是交朋友，員工好不好相處是其次，如果辦了員工旅遊，在某段時間上，就要強迫某些人配合其他人的生活習慣，或是其他家屬的生活習慣，甚至影響到工作氣氛，這事能避免就避免。但在工餘時間，員工自行組織旅遊，敞公司採開放態度
  • 尾牙抽獎： 創辦人討厭看到沒中獎或是中小獎的人的表情，這事能避免就避免，至於辦尾牙，要在上班時間一起吃個大餐，那也是可以的。惟什麼餘興節目，這事能避免就避免，創辦人討厭扮成猴子，也不想強迫員工花時間想餘興節目，敝公司聘請員工，主要目的是賺錢，不是演猴戲，又如果是額外請歌唱團體的話，創辦人是偏好鐵娘子，但就怕大家吃不下飯，所以這事能避免就避免

上表以任職 3 年時間來估算，是敝公司希望吸引聰明、熱情、認真且懶惰的工程師加入，無經驗者願意花時間培訓，就算工作績效達不到公司所給付的成本，只要不賠太多(標準自在創辦人心中)，敝公司願意提供 3 年的任職時間，而有經驗者則期待大家能一起合作 3 年。 3 年期滿，再依 3 年總績效評估，提供「期滿不再續用」、「協助創業」、「擔任敝公司相關企業董事」等選項。

  • 在「協助創業」上，主要提供 3 種協助：
    1. 納為下包廠商，每月契約金額為目前薪資福利總值以上
    2. 投資新公司資本額 10 %以下或投資金額為平均月薪資福利乘以 4 倍以下
    3. 無償提供新公司使用敝公司開發的辦公軟體
  • 在「擔任敝公司相關企業董事」上，其新的「薪資福利」將高於目前 2 倍以上，並另有無償員工配股

加班費： 敞公司原則上僅同意董事及輪班之系統管理員加班，加班費依勞動基準法計算。其餘員工非經董事同意不得加班。

員工所使用的 3C 及辦公設備皆為自選，任職滿 3 年後，即屬於員工所有。惟購置設備的支出模式尚無定案，待與員工討論。尚無定案的原因，主要是避免員工任職不久後，即告離職，留下一堆自選的 3C 及辦公設備給敝公司造成麻煩，畢竟其他新進員工也會有自選設備，而敝公司也不打算作二手用品賣家。

工作內容

敝公司目前主要營運模式為接案公司，提供業主完整的資訊系統開發、維運服務及任何難題諮詢。代管之系統除 Open Source 軟體外，餘為敝公司協助業主客製化之系統。目前代管系統如下：

  • 任務追蹤系統
  • 文件管理系統
  • 內容管理系統
  • 電商銷售系統
  • 電子報收發系統
  • 信用卡支付系統

伺服器皆置於雲端平台，主要是 Amazon AWS 及 Google App Engine 。

工作角色分為三大部份： 系統管理員(DevOps)、後端程式開發員、前端設計師。每一員工皆需擔任此三角色，也就是所謂的「全能工班(營建業術語: Full-Capacity Operation) || 網站全端工程師(軟體業術語: Full Stack Web Engineer)」，惟其工作比例可有不一，目前重點徵求後端程式開發員，內容比例約為 10% DevOps 、 70% 後端開發 、 20% 前端開發。

在溝通語言及文件寫作上，主要使用中華民國國語，次要使用英文，少部份使用日文。

以下為工作時所常運用之工具，但視未來業務發展得擴展下列清單。簡單講，到何六百工作就是到何六百學習，而學海無崖：

  • AWS: EC2/S3/CloudFront/RDS/VPC/Route 53/IAM/SES/SNS/Lambda/API Gateway/DynamoDB
  • GAE: Python
  • Ubuntu Linux
  • pfSense OS: OpenVPN/IPSec
  • NGINX
  • uWSGI
  • PostgreSQL
  • Jenkins / Buildbot
  • Python2 + Django + south + tastypei + guardian + mediagenerator + virtualenv
  • git / hg
  • GnuPG
  • HTML / CSS
  • Bootstrap
  • javascript + jQuery
  • Sikuli X / Selenium

以上工具，創辦人並不是從大學教育中所習得，所以十分了解學歷與上述工具的相關性，招募員工完全不會考慮員工的最高學歷及就學科系為何? 敝公司在乎的是聰明、熱情、認真且懶惰的工程師。

敞公司的管理哲學為管事不管人，一切實事求是。主要原因是創辦人是懂事不會懂人，怕把人給管壞了。公司組織架構採用古老的魚網式組織及以前很流行的學習型組織，階層只有兩級： 員工兼董事、員工。所以每一位員工皆有機會擔任專案經理，就專案範圍來說，專案經理最大。請不要把專案經理跟業務經理搞混了，敞公司目前的業務經理、會計經理、人資經理、打雜經理都是創辦人。

若您實在不了解專案經理及專案管理到底是什麼東西? 恰恰好，創辦人表定上的學歷文件說明了他在這個領域花了很多時間學習及思索，來吧! 就來何六百與創辦人一起學習、思考吧!

總結來說，到何六百工作，您有時候會是系統管理員、有時候是後端程式開發員、有時候是前端設計師、有時候多兼個專案經理，但不變的是每月工時為 160 小時以下。

彈性工時

依 104 年 12 月 16 日修正之新版勞基法第 30 條規定，敝公司之員工工作時數如下：

  • 每周工時最多為 48 小時，每月總工時最多為 160 小時
  • 單日上班時數上限為 8 小時，上班時間為早上 6 點至晚上 10 點的區間內任選時段，每個工作時段以實際勞務提供為算，最少須連續 3 小時，最長原則不得超過 4 小時，但如工作有連續性或緊急性時，最長不得超過8小時，工作時段之間的非工作時間/休息時間至少須 1 小時以 上。
  • 其他未盡事項，皆依勞工在事業場所外工作時間指導原則辦理。

公司願景

為滿足創辦人無法擔任律師的幻想，所以極度希望把本公司經營得像律師事務所一樣，收費照工作時數計算，每一位客戶就是一件獨立的專案，雖然喜愛開放原始碼，但也會對每一位客戶的營業秘密守口如瓶。

上述是一種偏執的幻想，不符學習型組織的共同願景勾勒方法。須待員工進來後共同討論。本段文句目前請視為空白。

招募限制

1.

應聘者須擁有中華民國國籍，遠端工作地點主要於中華民國實際管轄境內，且須具備或可申請 5M/sec 以上之網路(含3G/4G電信上網)。

2.

依我國就業服務法及外國人從事就業服務法規定，未擁有中華民國國籍者，敝公司目前無法聘雇。主要癥結在於敞公司資本額或營業額未達門檻，以及要求員工需在中華民國境內值班。

敞公司未來會逐步打破此兩項限制，也懇請有意任職敞公司的無本國籍朋友多留點心，或是留個 Email ，未來敞公司具有聘雇非本國籍員工資格時，會立即告知。

3.

應聘人禁止在筆試、電談(字談)透露個人識別資訊或利用其他管道向考試委員(也就是創辦人)關說。個人識別資訊包含但不僅限於電子信箱、部落格、網站、網域、學歷、科系、學校、種族、階級、宗教、黨派、籍貫、出生地、性別、性傾向、年齡、婚姻、身心障礙或以往工會會員身分者，容貌及五官則開放於面試過程展現並在此時檢驗有效之中華民國身份證正本。

應聘人無需準備履歷或自傳，考試委員對於您有幾個父親、幾個母親、個性如何…等無關敞公司賺錢的資訊一點也沒有興趣。展現您學習工具、運用工具的聰明、熱情、認真及 懶惰足已。

4.

敞公司招募員工在筆試、電談(字談)、面試上，會花費長時間，對單一求職者而言，短可能也要花到 1 個月以上，對當下需職孔急的待業人士而言，會是杯水車薪。建議求職者可在學校就讀(就算還有 4 年才畢業)或在其他企業任職時，即參加招募，筆試通過即無限期保留筆試成績。敞公司開發的筆試系統可完全保護您的身份，不會讓第二個人知道，除非您本人透露。而電談(字談)、面試可等到比較明朗的可上班日前附近再來舉行也是歡迎的。
